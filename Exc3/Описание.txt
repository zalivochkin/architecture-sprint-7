Способ взаимодействия с сервисами Интеллектуальный домофон и Интеллектуальный шлагбаум зависит от API этих сервисов, которое предоставляет партнер.

Если эти системы обеспечивают возможность передавать данные для аутентификации в систему PropDevelopment, то взаимодействие можно организовать следующим образом:
- для партнера заводится сервисный аккаунт
- используя этот аккаунт сервисы Интеллектуальный домофон и Интеллектуальный шлагбаум передают данные для аутентификации (биометрию или номер авто)
  и идентификатор объекта (двери или шлагбаума)
- сервис tenant-core-app производит аутентификацию и авторизацию и даёт ответ - открывать или нет

При таком взаимодействии за все вопросы безопасности будет отвечать система PropDevelopment и данные клиентов не будут передаваться третьим лицам.
Для защиты от DoS атак надо поставить RateLimiter.
Для защиты передаваемых данных надо использовать TLS актуальной версии.

Второй вариант - это если сервисы Интеллектуальный домофон и Интеллектуальный шлагбаум спроектированы так, что могут работать только со своей базой данных.
Тогда взаимодействие можно сделать так:
- если клиент желает использовать эти сервисы, то он даёт согласие на передачу своих персональных данных (в мобильном приложении)
- сервис tenant-core-app используя REST API сервиса партнера передаёт биометрию, номер авто клиента и список разрешённых объектов
- сервис партнера сохраняет эту информацию в своей БД и дальше использует без участия сервисов PropDevelopment

При этом варианте взаимодействия ответственность за хранение и обработку персональных данных будет лежать на партнере.
Для защиты передаваемых данных как и в первом варианте надо использовать TLS актуальной версии.

В случе если клиент даёт команду на открытие через приложение, то тогда в обоих случаях сервис tenant-core-app используя REST API сервиса партнера передаёт команду нужному объекту.
